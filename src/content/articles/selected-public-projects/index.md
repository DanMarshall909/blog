This page is a concise reference for public projects I am comfortable sharing with recruiters, hiring managers, and technical reviewers.

It is not a complete list of my work. Some of my strongest current AI and developer-tooling work is in private repositories because there is commercial potential or unfinished product direction involved. Where useful, I can grant direct access to specific private repositories for a technical reviewer.

## AI and developer tooling

### [tome](https://github.com/DanMarshall909/tome)

Python MCP server for semantically searching books, long-form documents, and C# source files.

This project is relevant to AI-assisted software engineering because it explores how long-form technical context can be indexed and retrieved for LLM workflows. The focus is practical semantic search over real developer material rather than a toy chatbot interface.

Relevant signals:

- Python
- MCP integration
- Semantic search
- Long-form document retrieval
- C# source-code context
- Local AI tooling

### [ai-rules](https://github.com/DanMarshall909/ai-rules)

My personal AI coding ruleset.

This repository captures the engineering constraints I use when working with AI coding tools. The goal is to make AI-assisted development more repeatable, safer, and easier to review by defining expectations around tests, scope control, code quality, and delivery workflow.

Relevant signals:

- AI-assisted development
- Coding guardrails
- Prompt and workflow design
- Test-led delivery
- Developer productivity
- Cross-tool AI workflow conventions

## C# engineering

### [DChess](https://github.com/DanMarshall909/DChess)

Chess engine written in C#.

This is a useful example of modelling a constrained rules domain in C#. Chess engines require careful handling of state, rules, legal moves, edge cases, and correctness. That makes the project a good signal for domain modelling and algorithmic problem solving.

Relevant signals:

- C#
- Domain modelling
- Rules engines
- Algorithmic problem solving
- State management
- Correctness-focused development

## Local AI experiments

### [dndan](https://github.com/DanMarshall909/dndan)

AI-powered roguelike game with old-school D&D rules, local LLM narration, image generation, containerisation, and Temporal orchestration.

This project is a work in progress. I would not present it as a finished product, but it is a useful example of how I experiment with orchestration, local AI, generated media, and game-like interaction loops.

Relevant signals:

- TypeScript
- Local LLM usage
- AI-generated narration
- Image generation
- Docker/containerisation
- Temporal orchestration
- Game systems
